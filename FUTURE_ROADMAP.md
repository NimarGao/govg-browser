# Govg Browser 未来演进规划蓝图（第四、五阶段）

随着第一、二、三阶段（包括 `WebContentsView` 架构重构、无痕模式隔离以及单站 Cookie 精细化管理）的完美收官，Govg Browser 已具备了现代桌面浏览器所需的扎实底座。

为了让 Govg Browser 进一步向功能完备、可扩展性强的生产力工具迈进，我们制定了后续的**第四阶段**（拓展生态与高级体验）与**第五阶段**（多维体验与安全增强）的升级蓝图。

## 实施方案细节

### 🚀 第四阶段：拓展生态与高级体验

此阶段旨在通过引入“可扩展生态”和“无缝数据迁移”解决从其他浏览器向 Govg Browser 迁移的阻力。

#### 1. 轻量化油猴脚本 (Userscript) 注入引擎
*   **具体改动**：
    *   在主进程增加 `userscripts` 模块，使用 `electron-store` 存储已安装脚本的元数据（包括 `@name`、`@match`、`@run-at` 等）。
    *   主进程每次在 `WebContentsView` 加载新网页时，根据配置匹配 URL，并通过 `executeJavaScript` 或在 preload 初始化阶段向页面注入脚本沙箱。
    *   提供一个简易的脚本配置页面（如 `swift://scripts`），支持添加、开启/关闭、卸载自定义脚本。

#### 2. 书签 Netscape 规范 HTML 导入与导出
*   **具体改动**：
    *   在书签侧边栏增加“导入/导出”按钮。
    *   导入时：弹窗选择本地 HTML 文件，主进程通过简单的正则或 HTML DOM 解析器提取出符合嵌套目录和 `<A HREF="...">` 标签结构的书签项，并合并去重导入到 `bookmarks` 数据库。
    *   导出时：格式化本地书签，生成符合标准 Netscape 规范的 HTML 文件供用户保存。

#### 3. 多线程下载器与状态控制
*   **具体改动**：
    *   使用类似 `got` 或原生 `http/https` 模块替换原生 `will-download` 控制逻辑，实现并发 Range 分片下载以提高下载速度。
    *   在下载列表中增加“暂停/恢复”按钮，并在主进程记录下载分片的 `tmp` 文件，以支持断点续传。
    *   下载完成通过 Windows 原生系统通知进行提示。

### 🛡️ 第五阶段：多维体验与安全增强

#### 1. 标签页拆分与多视窗 (Multi-Window)
*   **具体改动**：
    *   支持通过鼠标拖拽标签栏上的某个 Tab，移动到窗口外部时触发 `BrowserWindow` 的动态创建，实现真正的多视窗浏览器体验。

#### 2. SSL 安全证书详情面板
*   **具体改动**：
    *   监听 `WebContentsView` 上的 `did-start-navigation` 及 `page-title-updated`。
    *   主进程捕获网络证书（通过 `webContents.getCertificate()`），在渲染进程点击盾牌锁扣图标时，将证书指纹、发行方（如 Let's Encrypt）、加密算法套件及是否为安全无痕会话显示在毛玻璃弹窗中。

#### 3. 广告拦截 EasyList 远程订阅
*   **具体改动**：
    *   在系统设置中加入“广告拦截规则订阅”区域。
    *   提供输入框允许用户输入自定义订阅源（例如 AdGuard Chinese URL）。
    *   主进程在后台定期拉取这些 txt 文件，并在本地编译为规则集进行热替换。
